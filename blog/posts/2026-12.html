10/2 things I love-hate about frontend design

Just shipped my first proper frontend project. Five love-hate moments from the journey: pure CSS, SCSS, Grid, responsive, accessibility — the eternal tour.

  1. Pure CSS, rediscovered. For a while, every project demanded a framework: Tailwind, Bootstrap, CSS-in-JS. Then you remember that pure CSS works. Custom properties give you variables. Cascade is a feature, not a bug. The community is quietly coming back to it.
  2. SCSS, when it still pays off. Nesting, mixins, partials. SCSS was the answer to a problem CSS hadn't solved yet. Now, with custom properties and native CSS nesting, the question becomes: do I still need a build step? Sometimes yes, often no.
  3. CSS Grid: love the result, hate the curve. Once it clicks, you wonder how you ever lived without it. Getting there takes weeks of fighting auto vs 1fr, start vs stretch, gap rules that bend depending on layout mode. Steep learning. Payoff worth it.
  4. Responsive, still a subject. Phones, tablets, foldables, ultra-wide screens, browser zoom, system fonts, dark/light auto-switch. The viewport is more fragmented than ever. "Mobile-first" became "mobile-and-everything-else-first." No clean answer, only choices.
  5. Accessibility, the part we keep deferring. You know the keyboard nav, the aria, the contrast ratios. You also know the deadline is tomorrow. So you ship "good enough" and tell yourself you'll come back. The TODO list grows. The users who actually need accessibility keep waiting.

First real project online. Imperfect, glitchy on some browsers, accessible enough but not perfect. Shipping is its own kind of love-hate: the moment of going live and the immediate regret of seeing your own work in the wild. Both feed the next iteration.
